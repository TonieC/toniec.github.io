Overview

Tflows is a FlowBot-based scripting system for Discord bots. It allows bot behavior to be defined using a DSL passed into bot.command().

Scripts are interpreted line-by-line and executed at runtime inside a command context.

Installation

Install Tflows via pip:

shell
$ pip install tflows

Links: PyPI  ·  GitHub

Quick Start

A minimal working bot using Tflows:

python · main.py
import sys
import os
sys.path.append(os.path.abspath(os.path.join(os.path.dirname(__file__), "..")))

from tflows import FlowBot

bot = FlowBot(prefix="!")

bot.command(
  name="test",
  code="""
send Hello World $ping
"""
)

bot.run("BOT_TOKEN")

This registers a !test command that replies with Hello World followed by the bot's current latency.

FlowBot

FlowBot registers commands using a name + script body.

python
bot.command(
  name="test",
  code=""" ... """
)

Parameters

  • name - command trigger name (string)
  • code - DSL script block (string)

Execution

  • Command triggered via prefix
  • FlowBot finds matching command name
  • Code string is parsed line-by-line
  • Embed or text output is generated

DSL Commands

embed

Starts embed-building mode.

No arguments. Acts as a mode switch.

$title[TEXT]

  • TEXT - embed title string
  • Supports no nested parsing except variables like $ping

$desc[TEXT]

  • TEXT - embed description
  • Supports multiline text
  • Supports variables: $ping, $time()

$ping

  • Returns bot latency (ms)
  • No arguments

$time([mode])

  • No args → full date + time default format
  • 12h → 12-hour format
  • 24h → 24-hour format
  • nodate → time only
  • notime → date only
  • multiple flags → e.g. $time(nodate;24h)

Embed DSL

Embeds are created inside embed mode using directive commands.

tflows DSL
embed
$title[Hello]
$desc[Ping: $ping]

Supported Fields

  • $title[TEXT] → embed title
  • $desc[TEXT] → embed description
Embed is finalized automatically at end of script block execution.

Variables

Static Variables

  • $ping → latency in ms
  • $id → returns Discord user ID based on context

Function Variables

$image

  • $image → returns avatar image URL based on context

Modes

  • $image or $image(user) → sender avatar
  • $image(mention) → first mentioned user avatar
  • $image(act) → mention if present, otherwise sender avatar

Behavior

  • Defaults to the command author avatar if no argument is provided
  • If mention is used but no mention exists, falls back to author avatar
  • Always returns a direct Discord CDN image URL string

Examples

tflows DSL

$image
$image(user)
$image(mention)
$image(act)

$server

  • $server → returns server name

Modes

  • $server or $server(name) → guild name
  • $server(boost) → number of boosts
  • $server(boostlvl) → boost level

Behavior

  • Defaults to returning the server name
  • Uses Discord guild properties internally
  • Returns values as strings

Examples

tflows DSL

$server
$server(name)
$server(boost)
$server(boostlvl)

$membercount

  • $membercount → returns total members

Modes

  • $membercount or $membercount(all) → total members
  • $membercount(user) → human users only
  • $membercount(bots) → bots only

Behavior

  • Defaults to total member count
  • user filters non-bot members
  • bots filters bot accounts
  • Returns values as strings

Examples

tflows DSL

$membercount
$membercount(all)
$membercount(user)
$membercount(bots)

$time

  • $time() → dynamic time output

Examples

tflows DSL

$time()
$time(12h)
$time(nodate)
$time(nodate;24h)

$id

  • $id → returns Discord user ID based on context

Modes

  • $id or $id(user) → sender ID
  • $id(mention) → first mentioned user ID
  • $id(act) → mention if present, otherwise sender ID

Behavior

  • Defaults to the command author ID if no argument is provided
  • If mention is used but no mention exists, falls back to author ID
  • Always returns a Discord user ID as a string

Examples

tflows DSL

$id
$id(user)
$id(mention)
$id(act)
