Overview

Tflows is a FlowBot-based scripting system for Discord bots. It allows bot behavior to be defined using a DSL passed into bot.command().

Scripts are interpreted line-by-line and executed at runtime inside a command context.

FlowBot

FlowBot registers commands using a name + script body.


bot.command(
  name="test",
  code=""" ... """
)

Parameters

  • name - command trigger name (string)
  • code - DSL script block (string)

Execution

  • Command triggered via prefix
  • FlowBot finds matching command name
  • Code string is parsed line-by-line
  • Embed or text output is generated

DSL Commands

embed

Starts embed-building mode.

No arguments. Acts as a mode switch.

$title[TEXT]

  • TEXT - embed title string
  • Supports no nested parsing except variables like $ping

$desc[TEXT]

  • TEXT - embed description
  • Supports multiline text
  • Supports variables: $ping, $time()

$ping

  • Returns bot latency (ms)
  • No arguments

$time([mode])

  • No args → full date + time default format
  • 12h → 12-hour format
  • 24h → 24-hour format
  • nodate → time only
  • notime → date only
  • multiple flags → e.g. $time(nodate;24h)

Embed DSL

Embeds are created inside embed mode using directive commands. 


embed
$title[Hello]
$desc[Ping: $ping]

Supported Fields

  • $title[TEXT] → embed title
  • $desc[TEXT] → embed description
Embed is finalized automatically at end of script block execution.

Variables

Static Variables

  • $ping → latency in ms

Function Variables

  • $time() → dynamic time output

Examples


$time()
$time(12h)
$time(nodate)
$time(nodate;24h)

Examples

Full Command


bot.command(
  name="test",
  code="""
embed
$title[Tflow Time System Demo]
$desc[
Ping: $ping

Default: $time(nodate;24h)
No Date: $time(nodate)
No Time: $time(notime)
24h: $time(24h)
12h: $time(12h)
]
"""
)

Simple Output


Ping: $ping
Time: $time()
